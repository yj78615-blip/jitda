## 배운 것
- **Soft-delete 와 unique 제약의 상호작용**: `(seriesId, order)` unique 를 유지하려면 자동 번호는 반드시 soft-deleted 도 max 계산에 포함. 안 그러면 재사용 시도로 충돌. 일반적인 함정.
- **Dynamic 라우트 배포 검증**: 서버가 protected · CSR 라우트는 curl 로 200 안 나옴 (307 to auth). 라우트 등록 여부만 확인하려면 status 코드 (307 = 있음 vs 404 = 없음) 로 충분.
- **Discriminated union 으로 mixed state 관리**: 편집 UI 에서 기존/신규 이미지 섞을 때 `{kind: 'existing'} | {kind: 'staged'}` 로 나누면 각 경로에서 필요한 필드가 컴파일러가 강제 → 실수 방지.
