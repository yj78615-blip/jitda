## 무엇을 했나
- 작가 프로필 페이지 (`/@handle`) 라우트 구현 — middleware rewrite → `profile/page.tsx`
- `[@handle]`→`[handle]`→`profile/[handle]`→`profile/?handle=` 우회 (Windows bracket 폴더 이슈)
- `AuthorHeroActions` (팔로우/구독/후원 버튼+모달), `AuthorContent` (탭+시리즈/포스트 그리드) 컴포넌트 작성
- `UserNav`를 루트 레이아웃에 추가 — 모든 페이지에서 로그인 상태 표시
- 기존 페이지에서 중복 `UserNav` 제거 (page.tsx, profile/page.tsx)

## 잘된 점
- middleware rewrite + query parameter 우회로 Windows bracket 폴더 문제 회피 성공
- `UserNav`를 layout에 한 번 넣어 모든 페이지 일괄 적용
- mockup(author.html) 기반 디자인을 그대로 구현

## 아쉬운 점
- `[@handle]` parallel route slot, `[handle]` dynamic route, `profile/[handle]` 세 번의 접근 끝에 해결 — 초기에 Windows 이슈를 몰라서 오래 헤맴
- 서버 기동/종료 반복, PowerShell Start-Process 안정성 부족으로 디버깅에 시간 소모
- 36회 file modify로 변경 범위가 과도하게 산재

## 배운 것
- Windows에서 Next.js `[bracket]` 폴더명이 동적 라우트로 인식되지 않을 수 있음 → 미들웨어 리라이트 + query parameter로 회피
- Prisma `include: { tag: ... }` 패턴은 join table → tag relation 접근
- `Start-Job`이 `Start-Process`보다 Windows 백그라운드 서버 유지에 더 안정적
