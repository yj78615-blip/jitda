## 배운 것
- **Vercel Hobby/Pro serverless body 는 4.5 MiB 하드 캡** — 서버 프록시 업로드 파이프라인은 소용량 (아바타 등) 전용. 웹툰·영상 등은 반드시 presigned URL 이나 클라 direct upload.
- **Prisma 트랜잭션 default timeout 5s** — 순차 update 다수 있을 땐 `{ timeout: 30_000 }` 옵션이거나 `Promise.all` 병렬화 필수. Vercel serverless 15s 함수 timeout 과 다른 축.
- **`withErrors` fallback 은 MVP 진단 방해** — 프로덕션 안정화 전에는 `err.message` 를 응답에 포함해서 클라 로 원인 노출하는 게 시간을 크게 아낌. 나중에 되돌리기 (`ponytail:` 주석 남김).
- **CSP `img-src` 와 `connect-src` 는 별도 관리 필요** — 이미지 표시는 img-src, fetch/XHR/PUT 은 connect-src. 두 개 다 Supabase 도메인 화이트리스트.
- **Next.js App Router 서버 컴포넌트/라우트가 클라와 응답 shape 계약을 명시하지 않으면 언랩 미스가 조용히 지나감** — DTO 를 wrapper 없이 반환할지 `{episode: ...}` 로 감쌀지 규약을 API 명세에 명시해야.
- **사용자 화면 안내는 실제 스크린샷 요청이 우선** — 옛 UI 로 안내하면 신뢰 잃고 시간 낭비.
