01공통 규약

Base URL & 버전

  • Prefix: /api/v1
  • Production: https://api.jitda.com/api/v1
  • Staging: https://api.staging.jitda.com/api/v1
  • Local: http://localhost:3000/api/v1

인증 헤더

  • 모든 인증 필요 엔드포인트는 Authorization: Bearer <access_token> 헤더 필수
  • Access token 만료: 1시간. Refresh token 만료: 30일.
  • Access token은 JWT (payload에 sub, role, exp 포함)

Content-Type & 언어

  • 요청·응답 모두 application/json; charset=utf-8 (업로드 제외)
  • 언어 협상: Accept-Language: ko / en (기본 ko) — 에러 메시지 다국어화용

ID 규칙

모든 리소스 ID는 접두어가 있는 문자열. 예: usr_2fN1a, srs_v7QeK, epi_8mBnc.

접두어리소스접두어리소스
usr_Usersub_Subscription
srs_Seriestip_Tip
epi_Episodecmt_Comment
pst_Postntf_Notification
img_Imagerep_Report
tag_Tagpyt_Payout

타임스탬프 · 통화

  • 모든 시각은 ISO 8601 UTC 문자열 — "2026-07-15T14:32:10Z"
  • 금액은 정수 (최소 단위). KRW의 경우 원 단위 정수 — 4900 = ₩4,900. USD는 센트 — 499 = $4.99
  • 통화 코드는 별도 필드 currency: "KRW" | "USD"

페이지네이션 (커서 기반)

목록 엔드포인트는 cursor pagination 사용. 쿼리 파라미터:

  • limit: 페이지 크기 (기본 20, 최대 100)
  • cursor: 이전 응답의 next_cursor 값
  • sort: 엔드포인트별로 정의 (예: latest, popular)

응답 구조:

{
  "items": [ ... ],
  "next_cursor": "eyJpZCI6InNyc19hYmMifQ==",
  "has_more": true
}

에러 응답 형식

모든 4xx / 5xx 응답 본문:

{
  "error": {
    "code": "invalid_credentials",
    "message": "이메일 또는 비밀번호가 올바르지 않습니다.",
    "details": { "field": "password" }   // 선택
  }
}
HTTP대표 코드의미
400invalid_request요청 본문·파라미터 유효성 실패
401unauthorized토큰 없음/만료
403forbidden권한 없음(구독 안 함, 관리자 아님 등)
404not_found리소스 없음
409conflict중복(이메일 사용 중, 이미 팔로우 등)
422validation_failed비즈니스 룰 위반
429rate_limited레이트 리밋 초과
500internal_error서버 오류

인증 계층 (Auth 태그)

태그의미
PUBLIC로그인 없이 접근 가능
USER로그인 필수(모든 로그인 유저)
AUTHOR본인 자원(내 시리즈·정산 등)만 접근 가능
ADMIN관리자 role 필수

레이트 리밋 (초안)

  • 미인증: IP당 60 req/min
  • 인증: 유저당 300 req/min
  • 업로드: 유저당 10 req/min
  • 초과 시 429 반환 + Retry-After 헤더

03인증 (Auth)

POST /api/v1/auth/signup PUBLIC

이메일로 계정 생성. 이메일 인증 링크 발송.

Request Body
{
  "email": "user@example.com",
  "password": "Str0ng!Pass",
  "display_name": "김달빛",
  "handle": "kimdalbit"
}
201 Created
{
  "user": { /* User */ },
  "email_verification_sent": true
}
POST /api/v1/auth/login PUBLIC

이메일·비밀번호 로그인.

Request Body
{
  "email": "user@example.com",
  "password": "Str0ng!Pass"
}
200 OK
{
  "access_token": "eyJhbGciOi...",
  "refresh_token": "eyJhbGciOi...",
  "expires_in": 3600,
  "token_type": "Bearer",
  "user": { /* User */ }
}
POST /api/v1/auth/refresh PUBLIC

Refresh token으로 새 access token 발급.

Request Body
{
  "refresh_token": "eyJhbGciOi..."
}
200 OK
{
  "access_token": "eyJhbGciOi...",
  "expires_in": 3600,
  "token_type": "Bearer"
}
POST /api/v1/auth/logout USER

현재 refresh token 폐기. Access token은 자연 만료(1시간).

Request Body
{
  "refresh_token": "eyJhbGciOi..."
}
204 No Content
(no body)
POST /api/v1/auth/oauth/{provider}/callback PUBLIC

{provider} = google | apple. OAuth 인가 코드를 서버에 넘겨 로그인·최초 로그인이면 자동 회원가입.

Request Body
{
  "code": "4/0AY0e-g4...",
  "redirect_uri": "https://jitda.com/auth/callback",
  "handle_suggestion": "kimdalbit"       // 신규 유저용, 선택
}
200 OK
{
  "access_token": "...",
  "refresh_token": "...",
  "expires_in": 3600,
  "token_type": "Bearer",
  "user": { /* User */ },
  "is_new_user": false
}
POST /api/v1/auth/password/reset-request PUBLIC

비밀번호 재설정 이메일 발송.

Request Body
{ "email": "user@example.com" }
204 No Content
(항상 204 반환 — 이메일 존재 여부 노출 방지)
POST /api/v1/auth/password/reset-confirm PUBLIC

이메일로 받은 토큰과 새 비밀번호로 재설정.

Request Body
{
  "reset_token": "abc123...",
  "new_password": "N3wStr0ng!Pass"
}
204 No Content
(no body)
GET /api/v1/auth/me USER

현재 로그인한 유저 정보 (권한·설정 포함).

Request
(no body)
Header: Authorization: Bearer <token>
200 OK
{
  "user": { /* User */ },
  "email": "user@example.com",
  "email_verified": true,
  "is_adult_verified": false,
  "role": "user",                        // "user" | "admin"

계정 삭제 요청. 30일 유예 기간 후 영구 삭제, 활성 구독 자동 해지.

Request Body
{
  "password": "current-password",        // 소셜 로그인 유저는 생략
  "reason": "not_using"                  // 선택
}
200 OK
{
  "scheduled_deletion_at": "2026-08-14T05:20:00Z"
